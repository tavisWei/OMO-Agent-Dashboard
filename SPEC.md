# 需求文档 | OMO Agent Dashboard

## 项目概述

- **项目名称**: OMO Agent Dashboard
- **项目类型**: 独立Web应用（本地优先的全栈项目）
- **核心价值**: 一站式可视化管控 Oh My OpenCode (OMO) 多智能体系统，降低配置门槛，提升开发效率
- **目标用户**: 本地开发者，使用 OMO 进行多智能体协作编程
- **开发周期**: 4-6 周

---

## Constitution（项目宪章）

### 愿景 Vision

> **让每个开发者都能轻松掌控 AI 智能体团队**

五年后，OMO Agent Dashboard 成为 OMO 用户首选的管理平台：
- 一键安装，零配置启动
- 3秒内看到所有智能体状态
- 像管理团队一样管理 AI 智能体
- 成本透明，每一分钱都知道花在哪

**成功意味着什么？**
- 用户不再需要手动编辑 JSON 配置文件
- 智能体状态一目了然，问题实时发现
- 项目间智能体配置复用，效率提升 10x

### 价值观 Values

1. **简单优先**: 零配置启动，5分钟内上手 -> 复杂功能按需开启
2. **实时为王**: WebSocket 推送，延迟 < 100ms -> 不接受轮询带来的滞后感
3. **本地至上**: 所有数据存在本地 SQLite -> 不依赖云服务，尊重隐私
4. **性能极致**: 支持 100+ 智能体同时管理 -> 卡顿是不可接受的

### 约束 Constraints

- **技术约束**: 
  - 必须基于 Node.js + SQLite（本地优先）
  - 必须支持 WebSocket 实时通信
  - 必须读取 OMO 配置文件（`~/.config/opencode/oh-my-opencode.jsonc`）
- **安装约束**: 
  - 单命令安装（`npm install && npm start`）
  - 不需要 Docker/PostgreSQL/Redis
- **兼容性约束**: 
  - 支持 macOS/Linux/Windows (WSL2)
  - 兼容 OMO v3.x 配置格式

### 核心功能列表

#### P0 - 必须有（MVP）

| 功能 | 描述 | 验收标准 |

### 用户故事详述

#### US-001: 快速查看智能体状态

**角色**: OMO 开发者
**行为**: 打开 Dashboard，立即看到所有智能体状态
**价值**: 无需终端命令，一目了然掌握全局
**优先级**: P0
**故事点**: 3

**前置条件**:
- OMO 已安装并配置
- Dashboard 已启动

**主流程**:
1. 用户访问 Dashboard（默认 `http://localhost:3001`）
2. 系统自动连接 OMO 配置文件
3. 系统通过 WebSocket 订阅智能体状态
4. 页面显示所有智能体的卡片网格
5. 每个卡片显示：名称、状态(运行中/空闲/错误)、当前模型、心跳时间

**备选流程**:
- 连接失败：显示错误提示 + 重试按钮
- 无智能体：显示空状态引导页

**关联验收标准**:
- AC-001.1: 页面加载时间 < 2s
- AC-001.2: WebSocket 连接成功率 > 99%

---

#### US-002: 按项目分组管理智能体

**角色**: OMO 开发者
**行为**: 创建项目，将相关智能体分配到项目
**价值**: 不同项目的智能体配置互不干扰，便于管理
**优先级**: P0
**故事点**: 5

**前置条件**:
- 至少有一个智能体配置

**主流程**:
1. 用户点击「新建项目」按钮
2. 输入项目名称和描述
3. 从列表中选择要分配的智能体
4. 保存项目
5. 侧边栏显示项目列表

**备选流程**:
- 项目名重复：提示「项目名已存在」
- 删除项目：确认弹窗，保留智能体（不删除）

**关联验收标准**:
- AC-002.1: 项目创建 < 1s
- AC-002.2: 智能体归属正确，无遗漏

---

#### US-003: 可视化配置智能体模型

**角色**: OMO 开发者
**行为**: 点击智能体卡片，编辑模型参数
**价值**: 不再手动编辑 JSON，所见即所得
**优先级**: P0
**故事点**: 8

**前置条件**:
- 存在可编辑的智能体配置

**主流程**:
1. 点击智能体卡片右上角「编辑」按钮
2. 弹出配置面板
3. 修改模型（支持自动补全）
4. 调整 temperature（滑块 0-2，步进 0.1）
5. 调整 top_p（滑块 0-1，步进 0.05）
6. 设置 maxTokens（输入框，范围 1-100000）
7. 点击「保存」，配置立即生效
8. 自动更新 `oh-my-opencode.jsonc`

**备选流程**:
- 非法模型名：红色提示 + 推荐可用模型
- 保存失败：保留填写内容 + 错误提示

**关联验收标准**:
- AC-003.1: 配置保存后 1s 内生效
- AC-003.2: JSON 文件格式正确（可通过 `opencode doctor` 验证）

---

#### US-004: 任务看板管理

**角色**: OMO 开发者
**行为**: 创建任务，拖拽到不同状态列
**价值**: 可视化追踪智能体工作进度
**优先级**: P1
**故事点**: 8

**前置条件**:
- 至少创建一个项目

**主流程**:
1. 在项目视图中，点击「新建任务」
2. 输入任务名称和描述
3. 选择负责的智能体
4. 任务出现在「待办」列
5. 拖拽任务到「进行中」
6. 完成时拖拽到「已完成」

**Kanban 列定义**:
| 列 | 描述 |
|----|------|
| 待办 (Backlog) | 新建的任务 |
| 进行中 (In Progress) | 正在执行的任务 |
| 已完成 (Done) | 已完成的任务 |
| 失败 (Failed) | 执行失败的任务 |

**关联验收标准**:
- AC-004.1: 拖拽操作流畅，无卡顿（帧率 > 30fps）
- AC-004.2: 任务状态变更后 500ms 内同步到数据库

---

#### US-005: 实时成本追踪

**角色**: OMO 开发者
**行为**: 查看各智能体/项目的 Token 使用量和成本
**价值**: 控制 AI 消费，每一分钱清晰可见
**优先级**: P1
**故事点**: 5

**前置条件**:
- 系统配置了 API Key（用于成本计算）

**主流程**:
1. 点击侧边栏「成本统计」
2. 选择时间范围（今日/本周/本月/自定义）
3. 查看总览：总 Token、总成本
4. 按智能体查看明细
5. 按项目查看汇总
6. 支持导出 CSV

**关联验收标准**:
- AC-005.1: 成本数据延迟 < 5min
- AC-005.2: 支持主流模型定价（OpenAI/Anthropic/Google）

### 数据库设计

```sql
-- 项目表
CREATE TABLE projects (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL UNIQUE,
  description TEXT,
  created_at INTEGER NOT NULL,
  updated_at INTEGER NOT NULL
);

-- 智能体配置表（缓存 OMO 配置）
CREATE TABLE agents (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  project_id TEXT REFERENCES projects(id),
  model TEXT,
  temperature REAL DEFAULT 0.7,
  top_p REAL DEFAULT 1.0,
  max_tokens INTEGER,
  status TEXT DEFAULT 'idle', -- idle, running, error
  last_heartbeat INTEGER,
  config_path TEXT,
  created_at INTEGER NOT NULL,
  updated_at INTEGER NOT NULL
);

-- 任务表
CREATE TABLE tasks (
  id TEXT PRIMARY KEY,
  project_id TEXT REFERENCES projects(id),
  agent_id TEXT REFERENCES agents(id),
  title TEXT NOT NULL,
  description TEXT,
  status TEXT DEFAULT 'backlog', -- backlog, in_progress, done, failed
  position INTEGER DEFAULT 0,
  created_at INTEGER NOT NULL,
  updated_at INTEGER NOT NULL
);

-- 成本记录表
CREATE TABLE cost_records (
  id TEXT PRIMARY KEY,
  agent_id TEXT REFERENCES agents(id),
  model TEXT NOT NULL,
  input_tokens INTEGER DEFAULT 0,
  output_tokens INTEGER DEFAULT 0,
  cost REAL DEFAULT 0,
  recorded_at INTEGER NOT NULL
);

-- 活动日志表
CREATE TABLE activity_logs (
  id TEXT PRIMARY KEY,
  agent_id TEXT REFERENCES agents(id),
  action TEXT NOT NULL, -- started, stopped, error, config_changed
  details TEXT,
  created_at INTEGER NOT NULL
);
```
